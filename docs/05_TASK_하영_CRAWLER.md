# 하영 님 작업 가이드: 학식 메뉴 크롤러

## 1. 이번 작업의 목적

이번 작업은 학교 학식 페이지에서 여러 식당의 메뉴를 수집하고, 백엔드로 보낼 수 있는 형태로 정리하는 것입니다.

크롤러가 수집한 메뉴는 앱의 “오늘의 학식 화면”에서 사용됩니다.

---

## 2. 이 작업을 통해 배우는 것

이번 작업을 하면서 다음을 익히는 것이 목표입니다.

```text
- 크롤링 작업이 어떤 순서로 진행되는지
- 웹페이지 주소와 페이지 구조를 확인하는 방법
- 웹 페이지에서 필요한 데이터를 찾는 방법
- Google Apps Script로 외부 페이지를 요청하는 방법
- 가져온 데이터를 JSON 형태로 정리하는 방법
- 백엔드 API로 데이터를 전송하는 기본 흐름
- 크롤링 실패 상황을 확인하고 로그를 남기는 방법
- AI 코딩 에이전트에게 작은 단위로 작업을 요청하는 방법
- Git 브랜치를 만들어 안전하게 작업하는 방법
```

---

## 3. 작업 전에 읽을 문서

작업 전에 아래 문서를 읽습니다.

```text
1. docs/01_VIBE_CODING_GUIDE.md
2. docs/02_GIT_BRANCH_GUIDE.md
3. docs/03_CRAWLING_BASICS.md
4. docs/04_CRAWLER_GUIDE.md
```

`docs/03_CRAWLING_BASICS.md`는 크롤링을 처음 하는 사람을 위한 문서입니다.

코드를 작성하기 전에 반드시 먼저 읽습니다.

---

## 4. 작업 브랜치 만들기

main 브랜치에서 직접 작업하지 않습니다.

아래 명령어로 작업 브랜치를 만듭니다.

```bash
git switch main
git pull origin main
git switch -c feature/crawler-menu-ingestion
```

이미 브랜치를 만들어둔 상태라면 다음 명령어로 이동합니다.

```bash
git switch feature/crawler-menu-ingestion
```

---

## 5. 담당 기능

하영 님이 담당하는 기능은 다음입니다.

```text
- 학교 학식 페이지 주소 확인
- 학교 학식 페이지 구조 확인
- 여러 식당의 메뉴 위치 확인
- Google Apps Script로 메뉴 수집
- 수집 데이터를 정해진 JSON 구조로 정리
- 백엔드 크롤러 입력 API로 전송
- 크롤링 실패 시 사용할 샘플 데이터 준비
```

---

## 6. 작업 순서

아래 순서대로 작업합니다.

```text
1. 학식 메뉴가 있는 웹페이지 주소를 찾는다.
2. 브라우저에서 해당 페이지를 열어 메뉴가 보이는지 확인한다.
3. 날짜를 바꾸면 주소나 화면이 어떻게 바뀌는지 확인한다.
4. 개발자 도구로 식당명과 메뉴 텍스트가 HTML 안에 있는지 확인한다.
5. Apps Script로 페이지 내용을 가져온다.
6. 가져온 내용에 메뉴 텍스트가 포함되어 있는지 확인한다.
7. 필요한 메뉴 정보만 추출한다.
8. 정해진 JSON 형식으로 만든다.
9. 백엔드의 크롤러 입력 API로 전송한다.
10. 실패했을 때 로그가 남는지 확인한다.
```

한 번에 전체 크롤러를 완성하려고 하지 말고, 먼저 페이지 내용을 가져오는 것부터 확인합니다.

---

## 7. AI에게 요청할 때의 방향

AI에게는 한 번에 전체 자동화를 맡기지 말고, 작은 단위로 요청합니다.

좋은 작업 순서는 다음과 같습니다.

```text
1. Apps Script로 웹페이지 HTML 가져오기
2. HTML에서 특정 텍스트 찾기
3. 식당별 메뉴 데이터로 정리하기
4. JSON 형태로 만들기
5. 백엔드 API로 POST 요청 보내기
6. 로그 남기기
```

AI에게 요청한 뒤에는 반드시 실행 결과와 로그를 확인합니다.

아래 조건을 만족하면 이번 작업이 완료된 것으로 봅니다.

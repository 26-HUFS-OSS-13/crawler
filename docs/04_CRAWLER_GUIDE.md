# 크롤러 작업 가이드

이 문서는 크롤러 작업을 맡은 하영 님를 위한 설명 문서입니다.

크롤러는 앱이 보여줄 학식 메뉴 데이터를 준비하는 역할을 합니다.

---

## 1. 크롤러가 필요한 이유

앱에서 학식 메뉴를 보여주려면 먼저 학교 학식 페이지에서 메뉴 정보를 가져와야 합니다.

사용자가 앱을 열 때마다 직접 학교 페이지를 읽는 것이 아니라, 크롤러가 미리 메뉴를 수집해 백엔드에 저장합니다.

이후 앱은 백엔드에 저장된 메뉴를 조회합니다.

전체 흐름은 다음과 같습니다.

```text
학교 학식 페이지
→ Google Apps Script 크롤러
→ 백엔드 API
→ 앱 화면
```

---

## 2. Google Apps Script의 역할

Google Apps Script는 학교 학식 페이지에 접근해서 메뉴 정보를 가져오는 역할을 합니다.

이 프로젝트에서 Apps Script가 해야 할 일은 다음입니다.

```text
1. 학교 학식 페이지 요청
2. 식당별 메뉴 정보 찾기
3. 날짜, 식당명, 식사 구분, 메뉴 내용을 정리
4. 정해진 JSON 형태로 변환
5. 백엔드 API로 전송
```

Apps Script 안에서 AI 분석이나 영양정보 계산은 하지 않습니다.

---

## 3. 수집해야 하는 정보

크롤러가 수집해야 하는 정보는 다음입니다.

```text
- 식당명
- 날짜
- 식사 구분
- 메뉴 제목
- 원본 메뉴 문자열
- 크롤링 시각
```

메뉴를 예쁘게 해석하거나 요약할 필요는 없습니다.

학교 페이지에 있는 메뉴 텍스트를 가능한 한 그대로 가져오는 것이 중요합니다.

---

## 4. 식사 구분

식사 구분은 다음 중 하나로 정리합니다.

```text
breakfast
lunch
dinner
other
```

학교 페이지에서 조식, 중식, 석식이 명확히 구분되어 있으면 각각 매핑합니다.

구분이 애매한 경우에는 `other`를 사용합니다.

---

## 5. 크롤링 결과의 목표

크롤러의 최종 결과는 백엔드에 보낼 수 있는 메뉴 데이터 목록입니다.

형태는 대략 다음과 같습니다.

```json
{
  "source": "google-apps-script",
  "crawledAt": "2026-05-05T00:10:00Z",
  "menus": [
    {
      "cafeteriaName": "학생식당",
      "date": "2026-05-05",
      "mealType": "lunch",
      "title": "중식 A코스",
      "rawMenuText": "쌀밥, 제육볶음, 된장국, 김치"
    }
  ]
}
```

정확한 API 형식은 백엔드 명세를 따릅니다.

---

## 6. 크롤링이 실패할 수 있는 이유

크롤링은 다음 이유로 실패할 수 있습니다.

```text
- 학교 페이지 구조가 바뀜
- 특정 날짜에 메뉴가 없음
- 페이지 요청이 실패함
- 메뉴가 이미지로만 제공됨
- 식당별 구분이 불명확함
```

실패 가능성이 있기 때문에 로그를 남기는 것이 중요합니다.

---

## 7. 작업 완료 기준

크롤러 작업은 다음 기준으로 확인합니다.

```text
- 오늘 날짜의 메뉴를 가져올 수 있다.
- 여러 식당의 메뉴를 구분할 수 있다.
- 결과가 정해진 JSON 구조로 나온다.
- 백엔드로 데이터를 보낼 수 있다.
- 오류가 발생했을 때 확인할 수 있는 로그가 있다.
```

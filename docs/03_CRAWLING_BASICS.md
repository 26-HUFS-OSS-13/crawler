# 크롤링 기초 가이드

이 문서는 크롤링을 처음 해보는 팀원을 위한 설명 문서입니다.

크롤러 코드를 작성하기 전에, 먼저 “크롤링이 어떤 순서로 진행되는지”를 이해하는 것이 중요합니다.

---

## 1. 크롤링이란?

크롤링은 웹페이지에 있는 정보를 프로그램으로 가져오는 작업입니다.

사람이 브라우저로 학교 학식 페이지를 열어 메뉴를 확인하는 것처럼, 크롤러는 코드로 그 페이지에 접근해서 필요한 메뉴 정보를 가져옵니다.

이번 프로젝트에서는 학식 페이지에서 여러 식당의 메뉴를 가져와 백엔드로 보내는 것이 목표입니다.

```text
학교 학식 웹페이지
→ 크롤러가 페이지 내용 가져오기
→ 필요한 메뉴 정보만 추출
→ JSON 형태로 정리
→ 백엔드 API로 전송
```

---

## 2. 크롤링을 시작하기 전에 알아야 할 것

크롤링을 하려면 먼저 다음 정보를 알아야 합니다.

```text
1. 크롤링할 웹페이지 주소
2. 메뉴가 페이지 안에서 어디에 표시되는지
3. 식당이 여러 개라면 어떻게 구분되는지
4. 날짜별 메뉴를 어떻게 조회하는지
5. 조식, 중식, 석식이 어떻게 구분되는지
6. 페이지 내용이 HTML로 들어있는지, JavaScript로 나중에 불러오는지
```

코드를 쓰기 전에 브라우저에서 직접 페이지를 열어보고 구조를 확인해야 합니다.

---

## 3. 웹페이지 주소 확인하기

가장 먼저 해야 할 일은 학식 메뉴가 있는 페이지 주소를 찾는 것입니다.

예를 들어 브라우저 주소창에 다음과 같은 주소가 있을 수 있습니다.

```text
https://example.ac.kr/cafeteria/menu
```

이 주소를 크롤러가 요청하게 됩니다.

다만 날짜를 바꾸면 주소가 바뀌는 경우도 있습니다.

예시는 다음과 같습니다.

```text
https://example.ac.kr/cafeteria/menu?date=2026-05-05
```

이런 경우에는 날짜를 바꿔가며 요청할 수 있습니다.

---

## 4. 페이지에서 메뉴 위치 확인하기

웹페이지 주소를 찾은 뒤에는 메뉴가 페이지 안의 어디에 있는지 확인해야 합니다.

브라우저에서 보이는 화면만 보는 것이 아니라, 개발자 도구를 열어 HTML 구조를 확인합니다.

일반적인 확인 순서는 다음과 같습니다.

```text
1. 학식 페이지를 브라우저에서 연다.
2. 메뉴 텍스트 위에서 우클릭한다.
3. 검사 또는 Inspect를 누른다.
4. 메뉴가 어떤 HTML 태그 안에 있는지 확인한다.
5. 식당명, 날짜, 메뉴 텍스트가 어떤 구조로 묶여 있는지 확인한다.
```

예를 들어 메뉴가 아래와 같은 HTML에 들어있을 수 있습니다.

```html
<div class="cafeteria">
  <h3>학생식당</h3>
  <div class="meal lunch">
    <p>쌀밥, 제육볶음, 된장국, 김치</p>
  </div>
</div>
```

이 경우 크롤러는 `학생식당`이라는 식당명과 `쌀밥, 제육볶음, 된장국, 김치`라는 메뉴 텍스트를 가져오면 됩니다.

---

## 5. HTML에 메뉴가 바로 없는 경우

가끔 브라우저에는 메뉴가 보이는데, Apps Script로 HTML을 가져오면 메뉴가 보이지 않을 수 있습니다.

이런 경우는 페이지가 JavaScript로 나중에 데이터를 불러오는 구조일 수 있습니다.

이때는 개발자 도구의 Network 탭을 확인합니다.

확인할 내용은 다음과 같습니다.

```text
- 메뉴 데이터를 불러오는 API 요청이 있는지
- 응답이 HTML인지 JSON인지
- 날짜를 바꿀 때 어떤 요청이 새로 발생하는지
```

메뉴 데이터가 별도 API에서 JSON으로 내려온다면, HTML 페이지를 파싱하는 것보다 그 API 주소를 직접 호출하는 방식이 더 쉽습니다.

---

## 6. 크롤러가 최종적으로 만들어야 하는 데이터

크롤러는 메뉴를 화면에 예쁘게 보여주는 역할이 아닙니다.

크롤러의 목표는 학교 페이지에서 가져온 정보를 백엔드가 이해할 수 있는 구조로 정리하는 것입니다.

이번 프로젝트에서는 메뉴 하나를 대략 다음 정보로 정리합니다.

```text
- cafeteriaName: 식당명
- date: 메뉴 날짜
- mealType: breakfast, lunch, dinner, other 중 하나
- title: 메뉴 제목 또는 코스명
- rawMenuText: 원본 메뉴 문자열
```

예시는 다음과 같습니다.

```json
{
  "cafeteriaName": "학생식당",
  "date": "2026-05-05",
  "mealType": "lunch",
  "title": "중식 A코스",
  "rawMenuText": "쌀밥, 제육볶음, 된장국, 김치"
}
```

---

## 7. 크롤링 작업의 기본 순서

실제 작업은 아래 순서로 진행합니다.

```text
1. 학식 메뉴 페이지 주소를 찾는다.
2. 브라우저에서 페이지를 열어 메뉴가 보이는지 확인한다.
3. 개발자 도구로 메뉴가 HTML 안에 있는지 확인한다.
4. Google Apps Script로 페이지 내용을 가져온다.
5. 가져온 내용에서 식당명, 날짜, 메뉴 텍스트를 추출한다.
6. 추출한 데이터를 JSON 배열로 만든다.
7. 백엔드 API로 전송한다.
8. 실행 로그를 확인한다.
```

처음부터 완성된 크롤러를 만들려고 하지 말고, 아래처럼 단계별로 확인하는 것이 좋습니다.

```text
1단계: 페이지 내용을 가져오는지 확인
2단계: 메뉴 텍스트 일부를 찾을 수 있는지 확인
3단계: 식당별로 나눌 수 있는지 확인
4단계: JSON 구조로 만들 수 있는지 확인
5단계: 백엔드로 보낼 수 있는지 확인
```

---

## 8. 크롤링할 때 자주 생기는 문제

크롤링 작업에서는 다음 문제가 자주 생깁니다.

```text
- 주소가 잘못되어 페이지를 가져오지 못함
- 메뉴가 이미지로 되어 있어 텍스트를 추출할 수 없음
- 페이지가 JavaScript로 메뉴를 나중에 불러옴
- 날짜를 바꾸는 방식이 복잡함
- 식당별 구분이 HTML에서 명확하지 않음
- 특정 날짜에는 메뉴가 없음
```

문제가 생겼을 때는 “코드가 틀렸다”고 바로 생각하기보다, 페이지 구조를 다시 확인해야 합니다.

---

## 9. 막혔을 때 공유할 정보

크롤링에서 막히면 아래 정보를 정리해서 공유합니다.

```text
- 크롤링하려는 페이지 주소
- 브라우저에서 메뉴가 보이는지 여부
- Apps Script로 가져온 내용에 메뉴 텍스트가 포함되어 있는지 여부
- 식당명과 메뉴가 HTML에서 어떻게 보이는지
- 에러 메시지 또는 로그
```

이 정보를 함께 공유하면 팀원 A가 문제를 훨씬 빠르게 확인할 수 있습니다.

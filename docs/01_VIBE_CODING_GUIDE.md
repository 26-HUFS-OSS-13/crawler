# AI 코딩 에이전트를 활용한 작업 가이드

이 문서는 AI 코딩 에이전트를 처음 사용하는 팀원을 위한 가이드입니다.

이번 프로젝트에서는 AI를 적극적으로 활용해 개발합니다. 하지만 AI에게 모든 것을 한 번에 맡기는 방식이 아니라, 사람이 작업 목표를 정하고 AI가 그 구현을 도와주는 방식으로 진행합니다.

---

## 1. 바이브 코딩이란?

바이브 코딩은 AI 코딩 도구에게 자연어로 원하는 기능을 설명하고, AI가 생성한 코드를 사람이 확인하면서 프로젝트에 반영하는 개발 방식입니다.

AI가 코드를 대신 작성해줄 수 있지만, 작업의 목표를 정하고 결과를 확인하는 책임은 작업자에게 있습니다.

이번 프로젝트에서 중요한 태도는 다음과 같습니다.

```text
AI가 코드를 대신 짜주지만, 책임은 작업자가 가진다.
```

즉, AI에게 요청한 뒤에는 반드시 앱을 실행해보고, 내가 맡은 기능이 실제로 동작하는지 확인해야 합니다.

---

## 2. 기본 작업 순서

작업은 매번 아래 순서로 진행합니다.

```text
1. 내가 맡은 TASK 문서를 읽는다.
2. 관련 GUIDE 문서를 읽는다.
3. 현재 브랜치를 확인한다.
4. 새 작업 브랜치를 만든다.
5. AI에게 작은 단위의 작업을 요청한다.
6. AI가 수정한 내용을 확인한다.
7. 앱을 실행해서 화면을 확인한다.
8. 오류가 있으면 오류 메시지를 확인하고 수정한다.
9. 기능이 동작하면 커밋한다.
10. Pull Request를 만든다.
```

이 순서를 지키면 실수해도 되돌리기 쉽고, 팀원이 서로의 작업을 확인하기도 쉬워집니다.

---

## 3. AI에게 한 번에 너무 큰 작업을 시키지 않기

AI에게 큰 기능을 한 번에 맡기면 코드가 많이 바뀌고, 문제가 생겼을 때 원인을 찾기 어렵습니다.

좋지 않은 요청은 다음과 같습니다.

```text
오늘의 학식 화면, 상세 화면, 요약 화면을 전부 만들어줘.
```

이렇게 요청하면 AI가 여러 파일을 한꺼번에 수정할 수 있고, 기존 코드 구조도 크게 바뀔 수 있습니다.

좋은 방식은 작은 단위로 나누는 것입니다.

```text
먼저 메뉴 카드 위젯만 만들어줘.
```

```text
이제 식당별로 메뉴 카드를 묶어 보여주는 리스트 화면을 만들어줘.
```

```text
메뉴 카드를 누르면 상세 화면으로 이동하도록 연결해줘.
```

작업 단위가 작을수록 이해하기 쉽고, 오류가 생겨도 고치기 쉽습니다.

---

## 4. 에러가 났을 때 처리 방법

에러가 나는 것은 자연스러운 일입니다.

중요한 것은 에러를 무시하지 않고, 어떤 상황에서 어떤 에러가 발생했는지 확인하는 것입니다.

에러가 나면 아래 정보를 정리합니다.

```text
1. 어떤 명령어를 실행했는지
2. 어떤 화면에서 문제가 났는지
3. 터미널 또는 콘솔에 나온 에러 메시지
4. 방금 수정한 파일
5. 기대한 동작
6. 실제 동작
```

AI에게 다시 물어볼 때도 이 정보를 함께 전달하면 더 정확한 답변을 받을 수 있습니다.

---

## 5. AI가 만든 코드 확인하기

AI가 코드를 만들었다고 해서 바로 끝난 것이 아닙니다.

작업자는 최소한 다음을 확인해야 합니다.

```text
- 요청한 기능이 실제로 실행되는가?
- 앱 실행 중 오류가 없는가?
```

이번 프로젝트에서는 완벽한 코드 리뷰보다, 내가 맡은 기능이 정상적으로 보이고 동작하는지를 확인하는 것이 우선입니다.

---

## 6. 작업할 때 기억할 점

AI를 사용할 때 가장 중요한 것은 작은 단위로 요청하고, 결과를 직접 확인하는 것입니다.

이번 프로젝트에서 팀원들이 익히면 좋은 것은 다음입니다.

```text
- 기능을 작은 작업으로 나누는 법
- AI에게 명확하게 요청하는 법
- 오류 메시지를 보고 문제를 공유하는 법
- 브랜치를 만들어 안전하게 작업하는 법
- PR을 통해 팀원과 작업 결과를 공유하는 법
```

코드를 전부 직접 작성하지 않더라도, 이 흐름을 경험하는 것이 이번 프로젝트에서 중요한 학습 목표입니다.
